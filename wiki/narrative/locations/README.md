# Locations

World-building location pages. Each page describes one place in the story world: its sensory qualities, psychological mapping, symbolic meaning, and the rules in effect within it.

## Pages
- [LogosPrime.md](LogosPrime.md) — KW1, rigid control / suppressed chaos, AEGIS's home domain

## Connections
- Related to: `wiki/narrative/characters/` — characters have `requires:` links to their primary locations
- Related to: `wiki/narrative/chapters/` — chapters list active locations in their `locations:` field
- Related to: `wiki/knowledge/rules/` — locations reference rules that govern their physics/logic
- Related to: `wiki/narrative/themes/` — locations embody themes through environmental storytelling

## Routing Layer
Part of the **Narrative Layer** — see [`wiki/narrative/README.md`](../README.md) for loading rules and mutation constraints.

## Conventions
- Naming: `TitleCase.md` (e.g., `LogosPrime.md`, `MnemosyneArchipelago.md`)
- Temporal: Usually no — most locations exist across the full story. If a location changes significantly (is destroyed, transformed), create a new page scoped by `valid_from`/`valid_until`.
